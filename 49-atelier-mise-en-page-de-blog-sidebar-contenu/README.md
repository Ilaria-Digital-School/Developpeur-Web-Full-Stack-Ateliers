# Atelier : mise en page de blog (sidebar + contenu)

RAPPEL THÉORIQUE CSS Grid est un système de mise en page bidimensionnel qui permet de contrôler simultanément lignes et colonnes. Contrairement à Flexbox, qui est unidimensionnel (une ligne ou une colonne à la fois), Grid excelle pour structurer des layouts globaux de page. Le mental model à retenir : pensez votre page comme un tableau invisible sur lequel vous placez vos éléments dans des zones nommées. La propriété `grid-template-areas` est particulièrement puissante pour cela : elle vous permet de dessiner visuellement votre layout directement dans le CSS. Erreurs fréquentes : oublier de définir `grid-template-columns` en cohérence avec les zones déclarées dans `grid-template-areas`, confondre `gap` (espace entre les tracks) et `padding` (espace interne d'un élément), ou négliger les breakpoints responsives qui doivent faire passer la sidebar sous le contenu sur mobile. Mini-exemple : `.container { display: grid; grid-template-columns: 250px 1fr; grid-template-areas: 'sidebar main'; gap: 1.5rem; }` — ici, la sidebar occupe 250px fixes et le contenu principal prend le reste de l'espace disponible. Sur mobile, un media query redéfinira `grid-template-columns: 1fr` et `grid-template-areas: 'sidebar' 'main'` pour empiler les zones verticalement. --- ÉNONCÉ DE L'ATELIER Vous allez concevoir la mise en page complète d'un blog fictif comportant une sidebar et une zone de contenu principal. L'objectif est de mobiliser CSS Grid pour structurer le layout global, et d'utiliser Flexbox là où cela est pertinent (navigation, cards d'articles, etc.).
