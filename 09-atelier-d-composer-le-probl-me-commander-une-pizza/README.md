# Atelier : décomposer le problème « commander une pizza »

**Rappel théorique** La décomposition de problèmes (ou « problem decomposition ») est l'une des compétences fondamentales du développeur. Elle consiste à prendre un problème complexe et à le découper en sous-problèmes plus petits, indépendants, et résolvables un par un. Cette approche s'appelle aussi la pensée algorithmique : avant d'écrire une seule ligne de code, un bon développeur modélise mentalement les étapes, les conditions et les données impliquées. Trois erreurs fréquentes chez les débutants : (1) vouloir résoudre le problème entier d'un coup sans le découper, (2) sauter directement au code sans réfléchir à la logique, (3) oublier les cas limites (que se passe-t-il si la pizzeria est fermée ? si le client n'a pas d'argent ?). Mini-exemple : « préparer du café ». Au premier regard, c'est une action simple. Décomposée, elle devient : vérifier s'il y a du café disponible → vérifier si la machine est propre → ajouter de l'eau → ajouter le café → lancer la machine → attendre → servir. Chaque étape est une instruction précise, séquentielle, et chacune peut elle-même être décomposée davantage. Le mental model à retenir : un programme, c'est une série d'instructions que la machine exécute dans un ordre précis. Votre rôle de développeur commence par savoir décrire cet ordre, en langage humain, avant même de penser à la syntaxe. --- **Énoncé de l'atelier** Dans cet atelier, vous allez exercer votre capacité à décomposer un problème du quotidien en étapes logiques et structurées, exactement comme le ferait un développeur avant de coder...
