# Dessiner un organigramme simple

**Rappel théorique** Un organigramme (ou flowchart) est une représentation graphique d'un algorithme. Là où le pseudo-code décrit les étapes en langage naturel structuré, l'organigramme les visualise sous forme de formes géométriques reliées par des flèches. Chaque forme a une signification précise : un rectangle représente une action ou une instruction (ex. : 'Afficher le résultat'), un losange représente une décision ou une condition (ex. : 'Est-ce que x > 0 ?'), un ovale ou arrondi représente le début ou la fin du programme, et un parallélogramme représente une entrée ou une sortie de données. Le flux d'exécution se lit toujours de haut en bas et de gauche à droite, en suivant les flèches. Une erreur fréquente chez les débutants est de confondre une action et une décision, ou d'oublier de fermer les branches d'un losange (un losange doit toujours avoir deux sorties : 'Oui' et 'Non'). Une autre erreur classique est de ne pas indiquer de condition de fin, laissant le flux sans issue. Mini-exemple : pour vérifier si un nombre est positif, on dessine un ovale 'Début', puis un parallélogramme 'Lire x', puis un losange 'x > 0 ?', avec une branche 'Oui' vers un rectangle 'Afficher positif' et une branche 'Non' vers un rectangle 'Afficher non positif', les deux rejoignant un ovale 'Fin'.
