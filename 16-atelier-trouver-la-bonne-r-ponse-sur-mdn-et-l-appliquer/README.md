# Atelier : trouver la bonne réponse sur MDN et l’appliquer

**Rappel théorique** La documentation MDN Web Docs (Mozilla Developer Network) est la référence absolue pour tout développeur web. Elle couvre HTML, CSS et JavaScript avec précision, fiabilité et exemples concrets. Contrairement à StackOverflow qui propose des solutions contextuelles parfois obsolètes ou approximatives, MDN fournit la spécification exacte d'une propriété, d'un élément ou d'une méthode, avec sa syntaxe officielle, ses paramètres, ses valeurs possibles, sa compatibilité navigateur et des exemples interactifs. Le mental model à adopter est simple : MDN d'abord, toujours. Quand tu ne sais pas comment fonctionne une propriété CSS ou une balise HTML, tu vas sur MDN, tu lis la définition, tu regardes la syntaxe, tu étudies l'exemple, et tu l'appliques dans ton code. Ensuite seulement, si tu bloques sur un cas particulier, tu peux consulter StackOverflow ou d'autres ressources. Erreurs fréquentes des débutants : copier-coller un exemple sans lire la documentation ; confondre plusieurs propriétés similaires (ex. `display: flex` vs `display: grid`) ; ignorer la section « Compatibilité des navigateurs » ; ne pas lire les notes d'avertissement signalées sur MDN. Mini-exemple : tu cherches comment centrer un élément horizontalement en CSS. Sur MDN, tu recherches `margin`, tu lis que `margin: 0 auto` fonctionne sur un élément bloc avec une largeur définie. Tu l'appliques directement dans ton CSS en comprenant pourquoi ça fonctionne, pas seulement comment. --- **Énoncé de l'atelier** Cet atelier a pour but de te faire pratiquer la recherche documentaire sur MDN de façon autonome et de t'obliger à appliquer ce que tu trouves dans...
