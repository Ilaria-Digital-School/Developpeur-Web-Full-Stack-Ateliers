# Atelier : ajouter un .gitignore adapté à un projet HTML/CSS

**Rappel théorique** Un fichier .gitignore est un fichier texte placé à la racine d'un dépôt Git qui indique à Git quels fichiers ou dossiers il doit ignorer, c'est-à-dire ne jamais versionner. Chaque ligne du fichier contient un motif (pattern) : un nom de fichier, un dossier, ou une expression générique (wildcard). Par exemple, la ligne `node_modules/` empêche Git de suivre tout le dossier node_modules, et `*.log` ignore tous les fichiers avec l'extension .log. Le mental model essentiel : Git ne versionne que ce qui est utile pour reconstruire ou partager le projet. Tout ce qui est généré automatiquement (dossiers de dépendances, fichiers de build, fichiers temporaires) ou personnel (configurations locales de l'éditeur, fichiers secrets) n'a pas sa place dans l'historique Git. Un .gitignore bien rédigé protège le dépôt des « pollutions » inutiles. Erreurs fréquentes : oublier d'ajouter le .gitignore dès la création du dépôt (les fichiers déjà trackés ne sont pas ignorés rétroactivement), confondre « ignorer » et « supprimer » (Git ignore les fichiers non encore trackés mais ne supprime pas ce qui est déjà suivi), ou écrire des chemins avec des barres obliques incorrectes selon l'OS. Mini-exemple : pour un projet HTML/CSS simple, on voudra ignorer le dossier `.vscode/` (configuration VS Code locale) et le fichier `.DS_Store` (métadonnée macOS) car ces éléments n'apportent rien aux autres contributeurs du projet. --- **Énoncé de l'atelier** Dans cet atelier, vous allez mettre en pratique la création et la configuration d'un fichier .gitignore adapté à un projet HTML/CSS réel.
