# Atelier guidé : repérer les composants (éditeur, terminal, navigateur)

**Rappel théorique** Un environnement de développement est l'ensemble des outils qui permettent à un développeur d'écrire, tester et exécuter du code. Il repose sur trois composants fondamentaux que tout développeur doit savoir identifier et distinguer. L'**éditeur de code** (ou IDE) est l'endroit où vous écrivez votre code source. Il offre des fonctionnalités comme la coloration syntaxique, l'autocomplétion et la détection d'erreurs. VS Code est aujourd'hui le plus utilisé dans l'industrie. Le **terminal** (ou console, shell) est une interface texte qui permet de communiquer directement avec le système d'exploitation. C'est via le terminal que vous lancerez des serveurs, installerez des dépendances, ou exécuterez des scripts. Sur Windows, on parle de PowerShell ou Git Bash ; sur macOS/Linux, de Terminal ou Bash/Zsh. Le **navigateur web** est l'outil de rendu et de débogage côté client. Il interprète le HTML, CSS et JavaScript, et embarque des outils de développement (DevTools) accessibles via F12. C'est là que vous observerez le résultat visuel de votre travail. L'erreur fréquente du débutant est de confondre ces trois espaces ou de ne pas savoir lequel utiliser selon la situation. Mini-exemple : quand vous tapez `node app.js`, vous êtes dans le terminal ; quand vous modifiez `app.js`, vous êtes dans l'éditeur ; quand vous ouvrez `index.html` pour voir le rendu, vous êtes dans le navigateur. --- **Énoncé de l'atelier** Cet atelier a pour but de vous faire prendre en main concrètement les trois composants de votre environnement de développement : l'éditeur de code, le terminal et le navigateur.
