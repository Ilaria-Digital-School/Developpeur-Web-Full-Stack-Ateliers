# Atelier : Inversion de l'ordre des éléments avec flex-direction

**Rappel théorique** La propriété CSS `flex-direction` contrôle l'axe principal d'un conteneur Flexbox, c'est-à-dire la direction dans laquelle les éléments enfants (flex items) sont disposés. Elle accepte quatre valeurs : `row` (valeur par défaut, gauche vers droite), `row-reverse` (droite vers gauche), `column` (haut vers bas) et `column-reverse` (bas vers haut). Comprendre `flex-direction` est fondamental car elle redéfinit l'axe principal et l'axe croisé, ce qui impacte directement le comportement de `justify-content`, `align-items` et `align-self`. Mental model à retenir : imaginez une flèche directionnelle à l'intérieur de votre conteneur. `flex-direction` oriente cette flèche. Tout ce qui se passe sur l'axe principal (`justify-content`) suit cette flèche. Lorsque vous passez à `column`, la flèche pointe vers le bas, et `justify-content` agit alors verticalement. Erreurs fréquentes : confondre l'inversion visuelle avec `order` et l'inversion de flux avec `flex-direction: row-reverse`. Une autre erreur classique est d'oublier que `row-reverse` inverse aussi le point de départ de `justify-content: flex-start`, qui se retrouve visuellement à droite. Enfin, inverser la direction n'affecte pas l'ordre dans le DOM, ce qui a des implications importantes pour l'accessibilité. Mini-exemple : ```css .nav { display: flex; flex-direction: row-reverse; justify-content: flex-start; /* les items apparaissent à droite */ } ``` --- **Énoncé de l'atelier** Dans cet atelier, vous allez construire une interface composée de plusieurs sections de mise en page qui exploitent explicitement `flex-direction` et ses valeurs d'inversion. Vous partirez d'un fichier HTML/CSS fourni en structure de base (voir les ressources) ou que vous créerez vous-même, et vous devrez produire trois composants distincts démontrant votre maîtrise...
