# Atelier : implémenter un jeu d'images adaptatif performant

**Rappel théorique** Les images représentent souvent plus de 60 % du poids total d'une page web. Les optimiser pour le responsive design est donc une priorité absolue en développement front-end professionnel. Deux attributs HTML5 sont au cœur de cette optimisation : `srcset` et `sizes`. L'attribut `srcset` permet de proposer plusieurs versions d'une même image à des résolutions différentes, laissant le navigateur choisir la plus adaptée à l'écran de l'utilisateur. L'attribut `sizes` indique au navigateur quelle largeur d'affichage occupera l'image selon les breakpoints définis, afin qu'il sélectionne la source la plus pertinente avant même de télécharger quoi que ce soit. Le lazy loading (attribut `loading="lazy"`) reporte le chargement des images hors écran jusqu'à ce qu'elles soient proches du viewport, réduisant ainsi le temps de chargement initial. Erreurs fréquentes : omettre `sizes` en présence de `srcset` (le navigateur se rabat alors sur 100vw), fournir des images mal dimensionnées ou non compressées, et oublier l'attribut `alt` indispensable à l'accessibilité.
