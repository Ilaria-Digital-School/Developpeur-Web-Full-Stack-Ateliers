# Atelier : Utiliser time, code et abbr correctement

**Rappel théorique** Les balises ``, `` et `` sont trois balises inline sémantiques que vous avez découvertes dans le chapitre précédent. Elles servent à donner du sens à des contenus spécifiques plutôt que de simplement les afficher visuellement. `` permet d'encoder une date ou une heure lisible par les machines grâce à son attribut `datetime`. Sa valeur doit suivre le format ISO 8601 (ex. : `2024-03-15` pour le 15 mars 2024, `2024-03-15T14:30` pour une date et heure précise). L'erreur classique est d'écrire du texte lisible dans `datetime` au lieu du format normalisé, ou d'omettre complètement l'attribut. `` est réservé aux fragments de code informatique inline. Elle ne remplace pas `` pour les blocs de code multilignes. Erreur fréquente : utiliser `` pour styliser du texte technique non lié à du code réel. `` sert à marquer les abréviations et acronymes. L'attribut `title` doit toujours contenir l'intitulé complet (ex. : `HTML`). Erreur fréquente : oublier `title`, ou utiliser `` pour des termes qui ne sont pas de véritables abréviations. L'objectif est de démontrer que vous savez employer correctement les balises ``, `` et `` dans un contexte réaliste, en respectant leurs règles d'usage et leurs attributs obligatoires.
