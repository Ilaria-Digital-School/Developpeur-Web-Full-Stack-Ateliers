# Mini-projet : amélioration du site statique + README complet

## Présentation du projet Ce mini-projet est le point de convergence de tout ce que vous avez appris dans cette section d'introduction : débogage, recherche de documentation, Git et rédaction de README. Vous allez améliorer un site statique existant (ou en créer un simple si vous n'en avez pas encore) en corrigeant des bugs, en appliquant de bonnes pratiques, et en le documentant correctement avec un README professionnel. L'objectif est double : produire un code propre et fonctionnel, ET le présenter comme un développeur le ferait dans un contexte professionnel réel. Ce projet simule ce que l'on attend d'un développeur junior lors de sa première contribution à un projet. ## Contexte professionnel simulé Vous intégrez une équipe de développement en tant que junior. On vous confie un site statique (HTML/CSS) à améliorer. Votre mission : identifier et corriger les problèmes existants, améliorer l'interface, versionner votre travail avec Git, et rédiger un README complet pour que n'importe quel développeur puisse comprendre, installer et utiliser le projet sans vous demander d'explications. ## Compétences validées - Utiliser les DevTools du navigateur pour identifier et corriger des bugs d'affichage HTML/CSS - Effectuer des recherches efficaces sur MDN et StackOverflow et appliquer les solutions trouvées - Versionner son travail avec Git en utilisant des commits clairs et réguliers - Rédiger un README complet et professionnel en Markdown - Adopter une démarche de débogage méthodique (pas à pas) - Documenter ses choix techniques de manière concise et utile
