# Atelier : ajouter la liste des compétences dans le CV

**Rappel théorique** En HTML, les listes permettent de structurer des données de même nature de façon sémantiquement correcte. On distingue trois types : la liste non ordonnée () pour des éléments sans ordre particulier, la liste ordonnée () pour des étapes ou classements, et la liste de définitions () pour des paires terme/description. Chaque élément d'une liste ou est encapsulé dans une balise . La règle d'or : n'utilisez jamais des listes uniquement pour leur rendu visuel (indentation, puces), mais parce que vos données forment réellement une collection d'éléments de même nature. Erreur fréquente : imbriquer des ou des là où un simple / serait sémantiquement juste. Autre erreur classique : oublier de fermer les balises , ce qui génère un comportement imprévisible selon les navigateurs. Exemple minimal correct : HTML CSS JavaScript La balise est généralement le bon choix pour une liste de compétences, car l'ordre n'a pas d'importance fonctionnelle. Ce rappel vous servira de boussole tout au long de cet atelier. **Énoncé de l'atelier** Vous avez déjà créé votre CV sémantique lors du mini-projet précédent. Celui-ci contient a minima un en-tête avec votre nom et vos coordonnées, une section expérience ou formation, et une structure HTML propre respectant les bonnes pratiques vues en cours. L'objectif de cet atelier est d'enrichir ce CV en y intégrant une section « Compétences » rédigée avec les balises de listes HTML appropriées.
