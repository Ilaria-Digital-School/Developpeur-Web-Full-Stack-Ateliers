# Mini-projet : site statique versionné (env + Git)

## Présentation du projet Ce mini-projet est l'aboutissement de la section 'Premiers pas : environnement de développement et mentalité développeur'. Il s'agit de créer un site statique simple en HTML/CSS et de le versionner intégralement avec Git, en appliquant toutes les bonnes pratiques abordées jusqu'ici. Le projet consiste à concevoir une page web personnelle (portfolio minimaliste, page de présentation ou page vitrine fictive) et à en gérer l'historique de développement via Git, depuis l'initialisation du dépôt jusqu'à la livraison finale. ## Compétences validées - Initialiser et configurer un dépôt Git local - Structurer un projet web statique (HTML/CSS) - Rédiger des messages de commit clairs, conventionnels et significatifs - Utiliser les branches Git pour isoler des fonctionnalités - Créer et configurer un fichier .gitignore adapté - Consulter et interpréter l'historique des commits avec `git log` - Maintenir un flux de travail Git cohérent et professionnel ## Contexte professionnel simulé Vous intégrez une équipe de développement en tant que développeur junior. Votre premier ticket consiste à livrer une page statique versionnée, prête à être revue par un senior. La qualité de votre historique Git sera évaluée au même titre que le code produit.
