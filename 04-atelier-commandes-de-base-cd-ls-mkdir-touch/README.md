# Atelier : commandes de base (cd, ls, mkdir, touch)

**Rappel théorique** Le terminal est une interface textuelle qui te permet de communiquer directement avec ton système d'exploitation en tapant des commandes. Contrairement à un explorateur de fichiers graphique, le terminal est rapide, précis et incontournable dans le quotidien d'un développeur full stack : tu en auras besoin pour lancer des serveurs, gérer des fichiers de projet, utiliser Git, installer des dépendances et déployer des applications. Quatre commandes constituent la base absolue de la navigation et de la gestion de fichiers : - **ls** (list) : affiche le contenu du répertoire courant. Sur Windows PowerShell, l'équivalent est `dir`. - **cd** (change directory) : permet de naviguer entre les dossiers. `cd ..` remonte d'un niveau, `cd nom_dossier` descend dans un dossier. - **mkdir** (make directory) : crée un nouveau dossier. - **touch** : crée un fichier vide. Sur Windows (PowerShell), l'équivalent est `New-Item nom_fichier`. Erreurs fréquentes : oublier de vérifier dans quel dossier on se trouve avant d'exécuter une commande (utilise `pwd` pour le savoir), confondre `/` et `\` selon l'OS, ou taper un nom de dossier avec des espaces sans guillemets. Mini-exemple : pour créer un dossier `mon-projet` sur le bureau et y créer un fichier `index.html`, on ferait : `cd Desktop`, puis `mkdir mon-projet`, puis `cd mon-projet`, puis `touch index.html`, puis `ls` pour vérifier. --- **Énoncé de l'atelier** Cet atelier a pour but de te faire pratiquer la navigation et la manipulation de fichiers et dossiers exclusivement via le terminal, sans jamais utiliser l'explorateur de fichiers graphique de ton...
