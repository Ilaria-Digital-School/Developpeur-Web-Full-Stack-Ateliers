# Atelier : installer Git et configurer user.name / user.email

**Rappel théorique** Git est un système de contrôle de version distribué créé par Linus Torvalds en 2005. Son rôle est simple : tracer l'historique de toutes les modifications apportées à un projet, permettre de revenir en arrière en cas d'erreur, et faciliter la collaboration entre développeurs. Contrairement à une simple sauvegarde de fichier, Git enregistre des "instantanés" (commits) de votre projet à des moments précis, avec un message explicatif et l'identité de l'auteur. Avant de pouvoir utiliser Git, il faut l'installer sur votre machine et le configurer avec votre identité. Cette identité (nom et email) sera attachée à chacun de vos commits. C'est indispensable dans un contexte professionnel : quand vous travaillez en équipe, chaque commit doit être clairement attribué à son auteur. Les deux commandes fondamentales de configuration sont `git config --global user.name "Votre Nom"` et `git config --global user.email "votre@email.com"`. L'option `--global` signifie que cette configuration s'applique à tous vos projets Git sur cette machine. Erreurs fréquentes : oublier d'installer Git avant de tenter de l'utiliser (la commande `git` n'est pas reconnue dans le terminal), confondre `--global` et `--local` (local s'applique uniquement au dépôt courant), ou encore renseigner un email différent de celui utilisé sur GitHub/GitLab, ce qui empêche la liaison correcte de vos contributions à votre profil.
