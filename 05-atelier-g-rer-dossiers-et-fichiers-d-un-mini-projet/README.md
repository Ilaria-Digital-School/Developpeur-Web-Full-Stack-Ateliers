# Atelier : Gérer dossiers et fichiers d’un mini-projet

## Rappel théorique Le terminal est une interface textuelle qui permet de communiquer directement avec le système d'exploitation via des commandes. Contrairement à un explorateur de fichiers graphique, le terminal offre une précision et une rapidité inégalées pour créer, déplacer, renommer ou supprimer des fichiers et dossiers — des opérations quotidiennes pour tout développeur. Quelques concepts clés à garder en tête : le répertoire courant est l'endroit où vous vous trouvez dans l'arborescence (affiché dans le prompt ou accessible via `pwd`). La navigation se fait avec `cd` (change directory). `ls` liste le contenu d'un dossier. `mkdir` crée un dossier, `touch` crée un fichier vide, `rm` supprime un fichier et `rm -r` supprime un dossier et tout son contenu. Erreurs fréquentes : oublier de vérifier son répertoire courant avant d'exécuter une commande (toujours faire `pwd` en cas de doute), confondre `/` (racine) et `./` (dossier actuel), ou utiliser des espaces dans les noms de fichiers sans guillemets (préférez les tirets `-` ou underscores `_`). Mini-exemple : pour créer un projet `mon-site` contenant un dossier `css` et un fichier `index.html`, on exécuterait : `mkdir mon-site`, puis `cd mon-site`, puis `mkdir css`, puis `touch index.html`. Simple, mais c'est exactement ce que font les développeurs au quotidien avant même d'écrire une seule ligne de code. --- ## Énoncé de l'atelier Vous venez d'apprendre les commandes de base du terminal.
