# Observation : inspecter un site non-responsive

**Rappel théorique** Le responsive design désigne la capacité d'un site web à s'adapter automatiquement à la taille de l'écran de l'utilisateur, qu'il s'agisse d'un smartphone, d'une tablette ou d'un ordinateur de bureau. Pour y parvenir, on s'appuie sur trois piliers fondamentaux : le viewport (la zone d'affichage visible du navigateur), les breakpoints (des seuils de largeur en pixels à partir desquels la mise en page change) et la fluidité (utilisation d'unités relatives comme %, em, rem, vw plutôt que des valeurs fixes en px). Un site non-responsive, en revanche, a été conçu avec des largeurs fixes (par exemple width: 960px), sans media queries, sans balise meta viewport correctement configurée. Résultat : sur mobile, le contenu déborde, l'utilisateur doit zoomer et scroller horizontalement — une expérience catastrophique. L'erreur la plus fréquente chez les débutants est de confondre un site « qui s'affiche sur mobile » avec un site « responsive ». Un site peut techniquement s'afficher sur un écran de 375px sans pour autant avoir été conçu pour cela. Exemple concret : si un élément a width: 1200px dans le CSS et qu'aucune media query ne vient le redimensionner, sur un écran de 390px (iPhone 14), cet élément dépasse de 810px à droite, créant un scroll horizontal non voulu.
