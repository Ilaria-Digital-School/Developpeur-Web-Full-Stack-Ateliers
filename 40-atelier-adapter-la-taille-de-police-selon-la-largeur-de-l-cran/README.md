# Atelier : Adapter la taille de police selon la largeur de l’écran

**Rappel théorique** Le responsive design repose sur l'idée que le contenu doit s'adapter naturellement à la taille de l'écran de l'utilisateur. La typographie ne fait pas exception : une police de 16px lisible sur desktop peut paraître minuscule sur mobile ou excessivement grande sur un grand écran. Les media queries CSS permettent de cibler des plages de largeur d'écran (breakpoints) et d'y appliquer des styles différents. En approche mobile-first, on commence par définir les styles de base pour les petits écrans, puis on les surcharge progressivement via des media queries `min-width`. Une erreur fréquente chez les débutants est de multiplier les media queries sans cohérence, ou de définir des tailles de police en pixels fixes alors que les unités relatives (`rem`, `em`, `%`, `vw`) offrent bien plus de flexibilité. L'unité `vw` (viewport width) est particulièrement puissante pour la typographie fluide : `1vw` correspond à 1% de la largeur du viewport. On peut également combiner `vw` et `calc()` pour un rendu encore plus contrôlé. Exemple minimal : ```css /* Mobile-first : base */ body { font-size: 1rem; } h1 { font-size: 1.5rem; } /* Tablette */ @media (min-width: 768px) { h1 { font-size: 2rem; } } /* Desktop */ @media (min-width: 1200px) { h1 { font-size: 2.8rem; } } ``` Ce mental model simple — partir du petit, enrichir vers le grand — est au cœur de cet atelier. --- **Énoncé de l'atelier** Vous allez créer une page HTML/CSS autonome qui démontre l'adaptation de la typographie selon la largeur de l'écran....
