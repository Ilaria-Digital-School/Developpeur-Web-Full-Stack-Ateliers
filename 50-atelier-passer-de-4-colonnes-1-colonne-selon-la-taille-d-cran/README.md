# Atelier : passer de 4 colonnes à 1 colonne selon la taille d'écran

**Rappel théorique** Le design responsive consiste à adapter la mise en page d'une interface à différentes tailles d'écran, sans créer plusieurs fichiers HTML distincts. CSS Grid est particulièrement puissant pour cela : en combinant des Media Queries avec la propriété `grid-template-columns`, on peut modifier radicalement la structure d'une grille selon le viewport. Le principe fondamental est le suivant — on définit d'abord la mise en page pour les grands écrans (desktop-first) ou les petits écrans (mobile-first), puis on surcharge les règles CSS dans des blocs `@media` ciblés. Une erreur fréquente chez les débutants est d'oublier que les propriétés Grid définies dans un `@media` écrasent celles définies précédemment : il n'est pas nécessaire de tout redéfinir, seulement ce qui change. Une autre erreur courante est de confondre `min-width` (mobile-first) et `max-width` (desktop-first) dans les Media Queries. Mini-exemple : pour passer d'une grille à 4 colonnes (desktop) à 1 colonne (mobile), on peut écrire : ```css .grille { display: grid; grid-template-columns: repeat(4, 1fr); gap: 1rem; } @media (max-width: 768px) { .grille { grid-template-columns: 1fr; } } ``` Ce mécanisme simple est le cœur de tout layout responsive basé sur Grid. --- **Énoncé de l'atelier** Dans cet atelier, vous allez concevoir une galerie de cartes (cards) responsive qui s'affiche sur 4 colonnes en desktop, 2 colonnes en tablette, et 1 colonne sur mobile.
