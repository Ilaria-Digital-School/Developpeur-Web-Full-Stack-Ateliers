# Atelier : installation et première ouverture de VS Code

**Rappel théorique** Un environnement de développement est l'ensemble des outils logiciels qui permettent à un développeur d'écrire, tester et exécuter du code de manière efficace. Parmi ces outils, l'éditeur de code occupe une place centrale : c'est là que vous passerez la majorité de votre temps. Visual Studio Code (VS Code) est aujourd'hui l'éditeur le plus utilisé dans le monde du développement web. Il est gratuit, open source, léger, extensible via des extensions, et compatible avec tous les langages que vous allez apprendre tout au long de ce cursus (HTML, CSS, JavaScript, Python, etc.). Une erreur fréquente chez les débutants est de confondre VS Code avec Visual Studio (IDE complet de Microsoft, beaucoup plus lourd et orienté .NET). Ce sont deux produits distincts. VS Code est l'outil qu'on utilise ici. Mental model à retenir : VS Code est votre 'cockpit'. Comme un pilote ne décolle pas sans vérifier ses instruments, un développeur ne code pas sans configurer correctement son éditeur. Un éditeur bien installé et bien configuré, c'est du temps gagné sur chaque session de travail. Mini-exemple concret : une fois VS Code installé et ouvert, vous pouvez créer un fichier index.html, taper du code HTML et voir immédiatement la coloration syntaxique s'activer. C'est le premier signe que votre environnement fonctionne correctement. --- **Énoncé de l'atelier** Cet atelier a pour but de vous faire passer de zéro à un environnement de développement opérationnel.
