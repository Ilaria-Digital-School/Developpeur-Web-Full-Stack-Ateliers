# Pseudo-code d'un trajet quotidien

**Rappel théorique** Un algorithme est une suite d'instructions précises, ordonnées et non ambiguës, destinées à résoudre un problème ou accomplir une tâche. Le pseudo-code est une façon d'exprimer cet algorithme en langage naturel structuré, sans se préoccuper de la syntaxe d'un langage de programmation particulier. Il permet de modéliser la logique d'un programme avant de l'écrire en code réel. Les trois structures fondamentales du pseudo-code sont : la séquence (les instructions s'exécutent les unes après les autres dans l'ordre), la condition (SI ... ALORS ... SINON, qui permet de prendre des décisions selon une situation donnée) et la boucle (TANT QUE ou POUR, qui répète un bloc d'instructions). Un algorithme bien conçu commence toujours par les entrées (ce qu'on reçoit), décrit un traitement (ce qu'on fait), et produit des sorties (ce qu'on retourne). Erreurs fréquentes à éviter : oublier des cas particuliers (ex. : que se passe-t-il si le bus est en retard ?), écrire des instructions trop vagues (ex. : « se déplacer » au lieu de « monter dans le bus ligne 12 »), ou négliger l'ordre logique des étapes. Mini-exemple — Préparer un café : DÉBUT Remplir la cafetière d'eau Ajouter le café moulu dans le filtre Allumer la cafetière TANT QUE le café n'est pas prêt Attendre 30 secondes FIN TANT QUE Verser le café dans une tasse FIN Ce mini-exemple illustre une séquence avec une boucle d'attente.
