# Réécrire un algorithme désordonné

**Rappel théorique** Un algorithme est une suite d'instructions ordonnées, non ambiguës et finies, destinées à résoudre un problème précis. L'ordre des étapes est fondamental : une instruction exécutée trop tôt ou trop tard peut rendre l'algorithme incorrect, voire inutilisable. Le flux d'exécution linéaire signifie que les instructions se déroulent les unes après les autres, dans un ordre strict, du début jusqu'à la fin. En pseudo-code, chaque ligne représente une étape atomique et lisible, indépendante du langage de programmation. Une erreur fréquente chez les débutants est de confondre l'objectif final avec le chemin pour l'atteindre : on sait ce que l'on veut obtenir, mais on écrit les étapes dans un ordre incohérent (par exemple, servir un plat avant de le cuisiner). Une autre erreur classique est de définir une variable après l'avoir déjà utilisée, ou d'effectuer un calcul avant d'avoir collecté les données nécessaires. Mini-exemple : pour calculer la moyenne de deux notes, un algorithme désordonné pourrait écrire « Afficher la moyenne » avant même d'avoir défini les notes ou effectué la division. Le rôle du développeur est de détecter ces incohérences et de replacer chaque instruction à sa juste place.
