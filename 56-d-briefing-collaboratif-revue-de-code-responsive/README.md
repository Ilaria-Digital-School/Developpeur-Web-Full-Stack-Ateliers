# Débriefing collaboratif : revue de code responsive

**Rappel théorique** Le design responsive repose sur trois piliers fondamentaux : les media queries, Flexbox et CSS Grid. Une media query permet d'appliquer des styles conditionnellement selon la taille du viewport (`@media (min-width: 768px) { ... }`). L'approche mobile-first impose de styliser d'abord le contexte mobile, puis d'enrichir progressivement via des breakpoints croissants — c'est l'inverse du 'desktop-first' qui était la norme avant 2010. Flexbox excelle pour les layouts unidimensionnels (une ligne ou une colonne), tandis que CSS Grid s'impose pour les layouts bidimensionnels (lignes ET colonnes simultanément). Un mental model utile : imaginez le mobile comme un train sur une seule voie, et le desktop comme un carrefour à plusieurs voies — vous construisez d'abord la voie simple, puis vous ajoutez les embranchements. Erreurs fréquentes à éviter : oublier ``, utiliser des pixels fixes pour les largeurs au lieu de unités relatives (`%`, `vw`, `rem`), négliger les états `:focus` visibles, ou encore empiler des media queries contradictoires sans logique claire. Mini-exemple : `display: grid; grid-template-columns: 1fr;` sur mobile, puis `grid-template-columns: repeat(3, 1fr);` à partir de 992px — simple, lisible, efficace.
