# Atelier : inspecter une page HTML avec DevTools

RAPPEL THÉORIQUE Les DevTools (outils de développement) sont intégrés directement dans votre navigateur (Chrome, Firefox, Edge). Ils permettent d'inspecter, de modifier temporairement et de déboguer n'importe quelle page web en temps réel, sans toucher aux fichiers sources. C'est un outil quotidien pour tout développeur web. Le modèle mental à retenir : chaque page HTML est un arbre de nœuds appelé DOM (Document Object Model). Le navigateur lit le fichier HTML et construit cet arbre en mémoire. DevTools vous donne une fenêtre directe sur cet arbre. Toute modification faite dans DevTools est temporaire : elle disparaît au rechargement de la page. Vous ne cassez rien. Erreurs fréquentes des débutants : confondre le fichier HTML source (ce que vous écrivez) et le DOM (ce que le navigateur interprète et affiche) ; croire que modifier un élément dans DevTools modifie le fichier sur disque ; fermer DevTools au moindre message rouge sans lire ce qu'il dit. Mini-exemple : si vous faites clic droit sur le titre d'une page, puis "Inspecter", DevTools ouvre le panneau Elements directement positionné sur la balise HTML correspondante. Vous pouvez double-cliquer sur le texte dans le panneau et le modifier. Le changement est visible immédiatement dans le navigateur, mais rechargez la page : tout revient à l'état initial. --- ÉNONCÉ DE L'ATELIER Cet atelier a pour objectif de vous faire manipuler concrètement les DevTools du navigateur pour inspecter et comprendre la structure HTML d'une page web réelle. Vous allez naviguer dans le DOM, identifier des balises, lire leurs attributs,...
