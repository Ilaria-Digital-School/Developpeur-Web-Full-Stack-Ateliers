# Atelier : section « À propos » adaptative

**Rappel théorique** Avant de plonger dans l'atelier, rappelons quelques fondamentaux essentiels. Le design responsive consiste à concevoir des interfaces qui s'adaptent fluidement à toutes les tailles d'écran, sans casser la mise en page ni dégrader l'expérience utilisateur. Flexbox est votre outil principal pour orchestrer l'alignement et la distribution d'éléments dans un conteneur, sur un axe principal (row ou column). La propriété flex-direction contrôle cet axe, flex-wrap gère le passage à la ligne, et les propriétés align-items / justify-content pilotent l'alignement sur les deux axes. Le mental model clé à retenir : pensez toujours en termes de conteneur et d'enfants. Le conteneur Flexbox distribue l'espace disponible ; les enfants s'y adaptent. Combinez ensuite les Media Queries CSS pour modifier le comportement Flexbox selon les breakpoints (typiquement : mobile < 768px, tablette 768–1024px, desktop > 1024px). Erreurs fréquentes à éviter : oublier de définir display: flex sur le conteneur parent, utiliser des largeurs fixes en pixels au lieu de valeurs relatives (%, em, rem), négliger les états intermédiaires (tablette), et omettre le meta viewport dans le HTML. Mini-exemple : une section avec une image et un texte côte à côte sur desktop, empilés verticalement sur mobile, se réalise ainsi : .about { display: flex; flex-wrap: wrap; gap: 2rem; align-items: center; } .about img { flex: 1 1 300px; max-width: 100%; } .about .text { flex: 1 1 300px; } @media (max-width: 768px) { .about { flex-direction: column; } } Ce pattern est exactement ce que vous allez construire dans cet atelier. ---...
