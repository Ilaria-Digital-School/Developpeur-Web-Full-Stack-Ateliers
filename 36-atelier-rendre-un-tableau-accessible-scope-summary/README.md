# Atelier : rendre un tableau accessible (scope, summary)

**Rappel théorique** Un tableau HTML est accessible lorsque les technologies d'assistance (lecteurs d'écran) peuvent interpréter correctement la relation entre les données et leurs en-têtes. Deux mécanismes sont essentiels pour cela. L'attribut `scope` s'applique sur les balises `` et indique si un en-tête concerne une colonne (`scope="col"`), une ligne (`scope="row"`), un groupe de colonnes (`scope="colgroup"`) ou un groupe de lignes (`scope="rowgroup"`). Sans `scope`, un lecteur d'écran ne sait pas si l'en-tête « Lundi » concerne la colonne entière ou uniquement la cellule adjacente. C'est l'erreur la plus fréquente. L'attribut `summary` (HTML4) était placé directement sur la balise `` pour décrire la structure du tableau aux utilisateurs non-voyants. En HTML5, il est déprécié et remplacé par deux alternatives : la balise `` (visible, recommandée) pour donner un titre au tableau, et un attribut `aria-describedby` pointant vers un paragraphe décrivant la structure complexe du tableau si nécessaire. Erreurs fréquentes : oublier `scope` sur tous les ``, utiliser `` à la place de `` pour les en-têtes, négliger ``, ou encore ne pas distinguer les en-têtes de ligne des en-têtes de colonne dans les tableaux à double entrée. Mini-exemple : `Matière` indique que cet en-tête s'applique à toute la colonne « Matière ». --- **Énoncé de l'atelier** Vous venez de construire un planning hebdomadaire lors de l'atelier précédent. L'objectif de cet atelier est de reprendre ce tableau (ou d'en construire un nouveau si vous n'avez pas conservé votre fichier) et de le rendre pleinement accessible selon les standards HTML5 actuels.
