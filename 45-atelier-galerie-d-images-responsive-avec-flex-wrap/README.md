# Atelier : galerie d'images responsive avec flex-wrap

**Rappel théorique** Flexbox est un modèle de mise en page unidimensionnel qui permet de distribuer l'espace entre les éléments d'un conteneur et de les aligner efficacement. Par défaut, un conteneur flex dispose ses enfants sur une seule ligne (ou colonne) sans jamais les faire passer à la ligne suivante. C'est la propriété `flex-wrap` qui modifie ce comportement : avec la valeur `wrap`, les éléments débordants sont renvoyés à la ligne suivante, et avec `wrap-reverse`, dans l'ordre inverse. La propriété raccourcie `flex-flow` combine `flex-direction` et `flex-wrap` en une seule déclaration (ex. : `flex-flow: row wrap`). Le mental model essentiel à retenir : un conteneur flex avec `flex-wrap: wrap` se comporte comme une grille souple. Chaque enfant occupe la place qu'on lui accorde (via `flex-basis`, `width` ou un pourcentage), et si la somme des largeurs dépasse celle du conteneur, un retour à la ligne automatique se produit. Combiné aux media queries et aux unités relatives (`%`, `rem`, `vw`), ce mécanisme permet de construire des mises en page véritablement responsives. Erreurs fréquentes : oublier de définir une largeur ou un `flex-basis` sur les enfants (ils restent alors tous sur une ligne), ne pas réinitialiser `box-sizing: border-box` (les paddings cassent les calculs de largeur), et négliger les gouttières (utiliser `gap` plutôt que des marges négatives).
