# Projet final de chapitre : tableau de bord adaptatif multi-breakpoints

## Description du projet Ce projet final de chapitre consiste à concevoir et développer un **tableau de bord (dashboard) adaptatif** complet, capable de s'afficher correctement sur trois environnements distincts : mobile (< 768px), tablette (768px – 1199px) et desktop (≥ 1200px). Ce tableau de bord simule une interface d'administration réaliste, intégrant l'ensemble des compétences acquises tout au long du chapitre : mobile-first, Flexbox, CSS Grid, media queries, accessibilité, performance des images et optimisation pour la production. ## Contexte professionnel Dans un contexte réel de développement Full Stack, le développeur front-end est fréquemment amené à livrer des interfaces d'administration ou de reporting accessibles depuis n'importe quel appareil. Ce projet simule une mission concrète : livrer un dashboard prêt pour la production, responsive, accessible et performant. ## Contenu du tableau de bord à réaliser Le tableau de bord devra inclure les composants suivants : 1. **En-tête (header)** : logo, titre de l'application, menu de navigation ou hamburger menu sur mobile. 2. **Barre latérale de navigation (sidebar)** : visible en colonne sur desktop, réduite ou transformée en menu bas/drawer sur mobile. 3. **Zone de statistiques (KPI cards)** : 4 cartes de métriques clés (ex. : chiffre d'affaires, nombre d'utilisateurs, taux de conversion, tickets ouverts) disposées en grille adaptative. 4. **Graphique ou visualisation de données** : une zone dédiée à un graphique (peut être simulé avec du HTML/CSS, pas obligatoirement une librairie JS). 5. **Tableau de données récent** : un tableau listant des entrées récentes (ex. : dernières commandes, derniers utilisateurs), scrollable horizontalement sur...
