# Atelier : refactorer une page existante en mobile-first

**Rappel théorique** L'approche mobile-first consiste à écrire le CSS de base pour les petits écrans en premier, puis à ajouter des media queries avec `min-width` pour enrichir progressivement la mise en page à mesure que la largeur d'écran augmente. C'est l'inverse du desktop-first, qui part d'un grand écran et réduit avec `max-width`. Le mental model essentiel est le suivant : par défaut, tout votre CSS s'applique au mobile ; chaque breakpoint est une couche d'amélioration, jamais une correction. Erreurs fréquentes : continuer à utiliser `max-width` dans une refactorisation mobile-first (ce qui signifie que vous n'avez pas réellement changé d'approche), empiler des media queries contradictoires, ou oublier de réinitialiser des propriétés inutiles sur mobile (marges larges, paddings excessifs, `display: grid` avec plusieurs colonnes dès le départ). Mini-exemple : au lieu d'écrire `.container { display: grid; grid-template-columns: repeat(4, 1fr); } @media (max-width: 768px) { .container { grid-template-columns: 1fr; } }`, vous écrivez `.container { display: grid; grid-template-columns: 1fr; } @media (min-width: 768px) { .container { grid-template-columns: repeat(4, 1fr); } }`. La logique est inversée, le mobile est la base. --- **Énoncé de l'atelier** Vous disposez d'une page HTML/CSS existante construite avec une approche desktop-first : la mise en page est pensée pour un grand écran et des media queries `max-width` tentent de l'adapter aux petits écrans, souvent de façon incomplète ou incohérente. Votre mission est de **refactorer intégralement cette page en adoptant une approche mobile-first rigoureuse**, en utilisant Flexbox et/ou CSS Grid selon les besoins de chaque section.
