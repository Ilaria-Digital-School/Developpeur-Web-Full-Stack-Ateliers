# Atelier : créer index.html et observer les modifications en direct

**Rappel théorique** Un fichier HTML est le point d'entrée de toute page web. L'extension .html indique au navigateur comment interpréter le contenu. Par convention, le fichier principal d'un site s'appelle index.html : c'est le fichier que le serveur (ou le navigateur en local) cherche en premier lorsqu'on accède à un répertoire. Le flux de travail fondamental du développeur web repose sur un cycle court et répétitif : modifier le code dans l'éditeur (VS Code), sauvegarder, puis observer le résultat dans le navigateur. Ce cycle peut être accéléré grâce à des outils comme Live Server (extension VS Code) qui rafraîchissent automatiquement la page à chaque sauvegarde. Sans cet outil, le rafraîchissement manuel (Ctrl+R / Cmd+R) reste la méthode de base. Erreurs fréquentes à éviter : oublier de sauvegarder avant de regarder le navigateur, nommer le fichier Index.HTML avec une majuscule (certains systèmes sont sensibles à la casse), ou ouvrir le fichier depuis un mauvais répertoire. Mental model clé : l'éditeur écrit, le navigateur lit. Ces deux outils sont distincts et ne se synchronisent pas magiquement — c'est vous qui contrôlez le flux. Mini-exemple : créer un dossier mon-projet/, y placer un fichier index.html contenant une balise h1, l'ouvrir dans le navigateur, modifier le texte dans VS Code, sauvegarder, puis constater le changement dans le navigateur. C'est exactement ce que vous allez faire dans cet atelier. --- **Énoncé de l'atelier** Vous venez de découvrir le flux de travail minimal éditeur → navigateur → rafraîchissement.
