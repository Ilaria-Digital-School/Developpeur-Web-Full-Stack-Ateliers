# Atelier : créer votre premier fichier index.html

**Rappel théorique** Un fichier HTML est un document texte structuré que le navigateur interprète pour afficher une page web. Chaque document HTML valide repose sur une structure minimale et obligatoire : la déclaration `` en toute première ligne, qui indique au navigateur que le document suit la norme HTML5 ; la balise `` qui encapsule l'intégralité du contenu ; la balise `` qui contient les métadonnées (titre de la page, encodage, liens vers les feuilles de style) ; et enfin la balise `` qui contient tout ce qui sera visible à l'écran par l'utilisateur. Le mental model à retenir : pensez à un document HTML comme à une boîte gigogne. La boîte extérieure est ``, à l'intérieur on trouve deux compartiments distincts — `` pour ce que le navigateur lit mais n'affiche pas directement, et `` pour ce que l'utilisateur voit. Erreurs fréquentes à éviter : oublier le `` (le navigateur entre alors en mode de compatibilité dégradé), ne pas fermer les balises ouvertes, ou placer du contenu visible dans le ``. Mini-exemple d'une structure minimale valide : ```html Ma première page Bonjour le monde ! ``` Ce rappel posé, passons à la pratique. --- **Présentation de l'atelier** Cet atelier a pour but de vous faire créer, de A à Z, votre premier fichier `index.html` fonctionnel, lisible dans un navigateur. Vous allez mobiliser tout ce que vous avez vu dans les activités précédentes : la structure minimale d'un document HTML, le rôle de chaque balise fondamentale, et les bonnes pratiques...
